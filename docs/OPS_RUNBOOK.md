# 运维手册（Runbook）

## 一、部署前检查
- Linux 服务器时间与时区正确
- 已安装 Docker / Docker Compose 或 Java 17 + Node 18 + MySQL 8
- 已准备上传目录与备份目录
- 已准备 HTTPS 证书

## 二、启动顺序
1. 启动 MySQL
2. 启动后端服务
3. 校验 Flyway 成功执行
4. 启动前端服务
5. 启动 Nginx 反向代理
6. 执行 `scripts/api_smoke_test.sh`

## 三、日常巡检
- 检查登录是否正常
- 检查上传目录是否可写
- 检查备份目录是否按计划产出文件
- 检查磁盘使用率
- 检查近 24 小时错误日志

## 四、故障处理
### 1. 无法登录
- 检查数据库连接
- 检查 JWT 密钥配置
- 检查管理员账号是否被锁定

### 2. 文件上传失败
- 检查磁盘空间
- 检查上传目录写权限
- 检查 Nginx 上传大小限制

### 3. 导出失败
- 检查服务内存
- 检查数据量是否超出当前机器规格
- 检查临时目录权限

### 4. 附件预览/下载失败
- 检查附件文件是否真实存在
- 检查数据库 file_path 是否一致
- 检查访问账号是否有权限

## 五、恢复流程建议
1. 停止应用写入
2. 备份当前异常数据
3. 恢复 MySQL 备份
4. 恢复附件目录
5. 启动服务
6. 执行冒烟脚本
7. 通知业务方回归验证
