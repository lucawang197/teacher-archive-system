# 安全基线建议（学校商用部署版）

## 1. 账号安全
- 默认密码首次登录强制修改
- 管理员账号不得复用演示密码
- 建议启用二次验证或至少限制后台访问 IP
- 连续登录失败 5 次后锁定 60 分钟

## 2. 权限控制
- 教师仅能访问本人档案、本人评分结果、本人附件
- 管理员拥有审核、评分、导出、用户管理、日志查看权限
- 所有导出与备份操作必须记录日志

## 3. 数据安全
- 生产环境数据库必须使用独立账号，不使用 root 作为应用连接账号
- JWT 密钥必须通过环境变量注入，不得写死在仓库
- 敏感个人信息导出需记录操作人与导出时间
- 备份文件目录应限制为运维与管理员可访问

## 4. 文件安全
- 仅允许 PDF/JPG/PNG
- 单文件上限 50MB，总请求上限 250MB
- 建议在生产环境对上传文件追加病毒扫描
- 附件下载/预览接口必须做权限校验

## 5. 网络安全
- 前端、后端统一走 HTTPS
- Nginx 限制请求体大小和连接超时
- 建议后台地址仅开放学校办公网/VPN
- 数据库禁止公网直连

## 6. 日志与审计
- 关键行为：登录、修改密码、上传档案、审核、评分、导出、备份、恢复
- 日志至少保留 180 天
- 建议每日归档，并接入监控告警

## 7. 灾备建议
- 数据库每日全量备份
- 附件目录每日打包增量备份
- 每季度至少做一次恢复演练
- 恢复演练应形成书面记录
